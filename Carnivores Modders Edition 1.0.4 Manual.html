Introduction

Thank you for downloading Carnivores Modders Edition. This patch aims to create a more flexible engine so mod-creators can build more detailed and varied mod packs than ever before.

Planned future features are listed here.

Change Log v1.0.4

  • Customisable snow/weather settings for each map
  • 32 Idle animation groups per creature
  • Set radar colour for creatures
  • Set blood colour for creatures
  • Option to disable bending for creatures
  • Option to disable/enable weaving for all creatures
  • Customise the weaving range for creatures
  • Option to make creatures aggressive only after being shot
  • Option to make creatures always flee after being shot
  • Option to make creatures flee if it hears gunshots regardless of aggression

Note: smell, hear and look in the RES have been renamed to smellK, hearK and lookK

Bug Fixes v1.0.4

  • Drowning and poison fog no longer crash the game.

All known bugs for 1.0.4 are listed here.

Getting Started

Before starting the game, complete the following steps:

  1. Installation

    To install Carnivores Modders Edition, you must have a vanilla copy of Carnivores 2 (always make a backup first!). Copy and paste the contents of the Carnivores 2 folder onto your vanilla copy of the game overwriting any existing files.

  2. Map Ambient Demo 2

    The Map Ambient Demo is a showcase of some of the features used in Carnivores Modders Edition including:

    • Pack hunting velociraptors.
    • Rare-spawning huntable Pachycephalosaurus.
    • Rare-spawning huntable Triceratops.
    • Ambient Icthyornis (spawns in flocks of around the coast).
    • Ambient Mosasaurus that spawns along the coast of Delpheous Hills.
    • Ambient Climatius (fish that spawns in deep ocean on Delpheous Hills).
    • Ambient Parexus (fish that spawns in the shallows around the estuary on the west side of Delpheous Hills).
    • Ambient Pink Parexus (fish that spawns in the pond on the northeast side of Delpheous Hills).

    To install the map ambient demo, copy the contents of the "Map Ambient Demo 2\HUNTDAT" folder onto your copy of Carnivores Modders Edition and overwriting any existing files.

    Note: Ichthyornis & Brachiosaurus will be updated with more animations/sounds in the next update and currently use some placeholder animations.

  3. Compatibility Mode

    Right click on CARN2.exe and click properties. On the compatability tab, check the box "Run this program in compatability mode" and select "Windows 98", then click ok.

  4. Driver Settings

    In the game settings, the audio driver must be set to "Direct Sound 3D", and graphics must be set to "Software" or "3DFX Glide" (this will be set automatically).

    If using 3DFX Glide (recommended), you will need to install nGlide if you haven't done so already - this is a one-time install so you won't need to repeat this when installing future versions of Carnivores Modders Edition.

    Download and install the latest version of nGlide here.

    Once installed, navigate to C:\ProgramData\Microsoft\Windows\Start Menu\Programs\nGlide and open the nGlide Configurator.

    Make sure the settings match those shown here, then click apply.

    You may also need to adjust the resolution in the game settings.

  5. Error fixes:

    If encounter this error on 3DFX Graphics, download and install the Visual C++ Redistributable (x64) here.

Short Guide to the AI

The _RES Sections

The _RES now contains the following new sections:

  • hunterinfo

    Use this section to define the hunter _RES info only.

  • oldambients

    The oldambients section must include 5 slots (ai 1-5) and can only use clone 1-6. Around 6 animals (depending on the density setting) spawn within the hunters view range, and are moved forward as the hunter moves across the map. If killed, any classic ambients will respawn on the edge of the view range. Spawn/avoidance regions cannot be set for creatures in these slots. Pack hunting cannot be used on creatures in this section.

  • corpseambients

    Use this section to define ambient corpses using ai/clone 6-9.

  • characters

    This section is for huntable creatures using ai/clone 10+. 4 creatures using AI (not clone) 10-16 will spawn in randomly per hunt whether or not they're selected from the hunt menu. Spawn/avoidance regions cannot be set for creatures in this section, but pack hunting can be used.

  • mapambients

    Map ambients spawn rate can be set in the _RES, as which area they spawn on, where on the map they spawn, and what times of day they spawn. A map ambient's _RES info must include a SpawnInfo subsection to spawn in. The clone value must be used to set the AI (this can be set to any AI in the game including huntable and classic ambient AI). Each map can have up to 108 varieties of map ambient, and 256 creatures in total (including huntables and classic ambients).

The AI

  • Huntables

    These AI (clone 10 - 27) can be used in the creatures and mapambients sections. Herding, swimming and aggression can be toggled for all these AI.

  • T-Rex

    This AI can be used in the creatures and mapambients sections. Pack hunting and swimming can be toggled for this AI, but it cannot be made non-aggressive.

  • Moshops, Gallimimus, Dimetrodon and Pig

    These AI can be used in the oldambients and mapambients sections. Swimming can be toggled for all these AI, but herding can only be toggled for these AI when they are used in the mapambients section. These AI cannot be made aggressive.

  • Pteranodon and Dimorphodon

    These AI can be used in the oldambients and mapambients sections. Herding can only be toggled for these AI when they are used in the mapambients section, and they cannot be made aggressive.

  • Brachiosaurus

    This AI can only be used in the mapambients sections. This is the classic brachiosaurus AI that won't react to being shot, but does come with a death animation. Herding can be toggled for this AI but it cannot be made aggressive.

  • Dangerous Brachiosaurus

    This AI can only be used in the mapambients sections. Unlike the classic brachiosaurus, this AI reacts to being shot. Herding and aggression can be toggled for this AI.

  • Land (Dangerous) Brachiosaurus

    This AI can only be used in the mapambients sections. This AI can spawn on dry land and reacts to being shot. Herding and aggression can be toggled for this AI. You can also toggle whether it can wade into the water as with other Brachiosaurus AI.

  • Ichthyornis

    This AI can only be used in the mapambients sections. It can only spawn on water, and is able to swim, fly and walk on land. Herding can be toggled for this AI, but aggression and swimming cannot.

  • Fish and Mosasaurus

    These AI can only be used in the mapambients sections. These AI are aquatic, and can swim to the surface to play idle animations. Aggression and herding/pack hunting can be toggled for both these AI. Mosasaurus AI requires a jumping animation, and is designed to move more realistically for a larger creature. Particle effects for jumping and idle animations can be customised in the _RES.

_RES value: snow1

Data type: TRUE/FALSE

When set to true, snow/rain will be spawned on the map. When set to false (or not defined) no weather effects will be created.

Note: Setting this value to true will not automatically add Ice Age snow to the map unless the other values are properly configured, see the Ice Age snow preset above.

>

_RES value: snow2

Data type: TRUE/FALSE

When set to true, a second particle is added to each snowdrop which can be configured individually.

>

_RES value: vSpd

Data type: Whole Number

The vertical speed that snow particles fall.

>

_RES value: hSpd

Data type: Whole Number

The horizontal speed of snow particles.

Note: speed and direction is effected by the wind.

>

_RES value: dens

Data type: Whole Number

The number of snow particles rendered at any time.

Note: cannot exceed 16000.

>

_RES value: red1/red2, gre1/gre2, blu1/blu2

Data type: Whole Number

Sets the colour of snow particles (RGB 0-255).
Only use red2, gre2 & blu2 to define the colour of the second particle if snow2 is set to TRUE.

>

_RES value: alp1/alp2

Data type: Whole Number

Sets the opacity of snow particles (0-255).
Only use alp2 to define the opacity of second particle if snow2 is set to TRUE.

>

_RES value: rad1/rad2

Data type: Whole Number

Sets the radius of snow particles.
Only use rad2 to define the radius of the second particle if snow2 is set to TRUE.

Data type: TRUE/FALSE

Required for dangerous brachiosaurus and land brachiosaurus AI only. This value controls sensitivity to obstacles and terrain. Increasing this value lowers the maximum gradient the brachiosaurus can ascend while running. When set to 0, the brachiosaurus will ignore all terrain. 128 is recommended for brachiosaurus, but if you place a smaller creature on brachiosaurus AI, then this value should be set to 64.

Note: steep underwater gradients like the northeast corner of Delpheous Hills can cause dangerous/land brachiosaurus to run into deeper water. Be mindful when designing maps, or include avoidance regions.

_RES value: anim

Data type: Whole Number

Position of an idle animation in the CAR file (starting from 0).

Each idlegroup can have up to 32 animations. These will be played in a random sequence.

>

_RES value: startChance

Data type: Decimal Number

The probability (after every walk cycle) that the idle group will start playing.

The value should be between 0 (never) and 1 (constant).

>

_RES value: endChance

Data type: Decimal Number

The probability that the idle group will stop playing. This probability will only be checked after playing the last-defined idle animation in the group or, if randomEnding is TRUE, after playing any idle animation in the group.

The value should be between 0 (never) and 1 (always).

>

_RES value: randomStarting

Data type: TRUE/FALSE

If set to true, then the first idle animation in the group will be randomised. Otherwise the first animation played in the group will always be the first defined-animation in the group.

>

_RES value: randomEnding

Data type: TRUE/FALSE

If set to true, then the idle sequence has a chance to end after each animation. Otherwise the sequence only has a chance to end after playing the last-defined idle animation in the group.

>

_RES value: instantRepeat

Data type: TRUE/FALSE

If set to true, then there is a chance for another idle sequence to start immediately after the idle group ends. Otherwise the creature will play at least one walk cycle before it has a chance to enter another idle sequence.

>

Data type: TRUE/FALSE

>

_RES value: xmax, xmin, ymax, ymin

Data type: Whole Number

Spawn regions are square. These four values control the four corners of the spawn region (xmin,ymin is the top left corner, xmax,ymax is the bottom right corner). These values are optional. Excluding these four values from a spawninfo subsection will result in the creature spawning over the entire map.

Note: It's recommended to open the map in Alt Editor 2, and use the mouse cursor to find the coordinates for these values.

Note: Remember that brachiosaurus, land brachiosaurus and ichthyornis AI creatures spawn regions must have sufficient shallow water. For fish/mosasaurus AI, make sure that spawn regions have water depth between mindepth and maxdepth.

_RES Subsection: tropinfo

Defines the position of a creature's trophy in the trophy room.

This subsection can be defined multiple times per creature for each trophy position. Each creature can have up to 32 trophy positions.

Note: Currently, the trophy room can only hold 128 trophies total.

Note: Currently, new trophies will not replace existing trophies if all the trophy positions are full.

Note: trophy = TRUE is also required (outside the tropinfo subsection).

Water Particle Effects

Fish/mosasaurus AI only. Water particle effects are used to make splashes when a mosasaurus AI creature jumps from the water, or when a fish or mosasaurus comes to the surface to play an idle animation. The particle values for the animation should be defined directly after an idleAnim or jumpAnim value. See the below example for the mosasaurus jumping and idle animation values.

These values set how particles are generated for each frame of the animation.

Note: Excessive use of particles could cause the game to lag or crash.

Note: Some particle values are affected by the creatures scale, so make sure to test thoroughly.

>

_RES value: jumpPartFrame1 / idlePartFrame1

Data type: Whole Number

The first frame of the animation (start counting from frame 0) where particles are generated.

>

_RES value: jumpPartFrame2 / idlePartFrame2

Data type: Whole Number

The last frame of the animation (start counting from frame 0) where particles are generated.

>

_RES value: jumpPartCnt / idlePartCnt

Data type: Whole Number

The number of particles generated per frame.

>

_RES value: jumpPartDist / idlePartDist

Data type: Whole Number

The maximum distance particles are generated from the creatures position.

>

_RES value: jumpPartMag / idlePartMag

Data type: Whole Number

The height of the particle splash.

>

_RES value: jumpPartOffset / idlePartOffset

Data type: Whole Number

Increasing this value shifts the particle effect forward towards the front of the creature.

>

_RES value: jumpPartAngled / idlePartAngled

Data type: TRUE/FALSE

Setting this to true will angle the splash particles to move forward with the creature, otherwise the splashes will shoot straight up.

>

_RES value: jumpPartCircle / idlePartCircle

Data type: TRUE/FALSE

Setting this is to true will add water circles for each particle generated.

Note: When the particles fall back into the water, they will also generate circles.
